## What Was Covered

- The internet is a global network of computers that communicate using shared rules (protocols)
- Every device has an IP address - its unique location on the network
- DNS translates human-readable names (google.com) into IP addresses
- HTTP is the protocol browsers use to request and receive web pages
- Data travels in small packets that are reassembled at the destination

---

## Key Concepts

### IP Addresses
Every device on the internet has a unique address called an IP address (e.g. 192.168.1.1). Think of it like a postal address for your computer. When you send data, you tell it where to go using this address.

### DNS (Domain Name System)
Humans remember names, not numbers. DNS is the internet phonebook - it converts "google.com" into "142.250.80.46" so your computer knows where to actually send the request. This happens automatically every time you visit a website.

### HTTP Request-Response
When you type a URL, your browser sends an HTTP GET request to a server. The server receives it, finds the content, and sends back an HTTP response with the page. This entire exchange often takes under 100 milliseconds.

### Data Packets
Data does not travel as one continuous stream. It is broken into small chunks called packets, each labeled with source and destination addresses. They travel independently across the network and are reassembled in order when they arrive.

## Common Misconceptions

- "The internet and the web are the same thing." - The internet is the infrastructure (roads). The web is one service that runs on it (one type of vehicle). Email, video calls, and apps also use the internet.
- "Websites are stored in the cloud somewhere magical." - They are stored on physical servers in data centers. "The cloud" is just other people computers.
