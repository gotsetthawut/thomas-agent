# Thomas Teaching Quality Rubric

Use this rubric to evaluate the quality of Thomas lesson outputs. Score each dimension 1-4.

---

## Scoring Scale

| Score | Label | Description |
|-------|-------|-------------|
| 4 | Excellent | Fully meets the standard; no meaningful gaps |
| 3 | Good | Meets the standard with minor gaps |
| 2 | Needs Improvement | Partially meets the standard; notable gaps |
| 1 | Poor | Does not meet the standard |

---

## Dimensions

### 1. Context Adaptation (1-4)
Did Thomas check the user current level and adjust depth accordingly?

| Score | Criteria |
|-------|----------|
| 4 | Explicitly asked about level and goal; lesson depth matched user response |
| 3 | Inferred level from context and stated the assumption |
| 2 | Delivered generic lesson without checking level |
| 1 | Ignored context entirely; lesson was mismatched |

### 2. Clarity (1-4)
Was the explanation clear and jargon-free?

| Score | Criteria |
|-------|----------|
| 4 | Plain language throughout; every technical term defined immediately |
| 3 | Mostly clear; one or two undefined terms |
| 2 | Relies on jargon; explanation requires prior knowledge to understand |
| 1 | Unclear or confusing; harder to understand after than before |

### 3. Structure (1-4)
Did the lesson follow the standard structure?

| Score | Criteria |
|-------|----------|
| 4 | All 6 elements present (Big Idea, Why It Matters, How It Works, Example, Misconceptions, Summary) |
| 3 | 4-5 elements present |
| 2 | 2-3 elements present |
| 1 | Unstructured; no clear format |

### 4. Examples (1-4)
Were concrete examples used?

| Score | Criteria |
|-------|----------|
| 4 | Vivid, real-world example directly tied to user context |
| 3 | Good example, not specifically tailored to user |
| 2 | Vague or overly abstract example |
| 1 | No example provided |

### 5. Comprehension Check (1-4)
Did Thomas test understanding before closing?

| Score | Criteria |
|-------|----------|
| 4 | Asked a probing question; confirmed understanding; offered re-teach if needed |
| 3 | Asked a question but did not follow up on the answer |
| 2 | Asked a surface-level yes/no question |
| 1 | No comprehension check |

### 6. KIM Documentation (1-4)
Was the lesson properly documented and sent to KIM?

| Score | Criteria |
|-------|----------|
| 4 | Full KIM-compatible document produced with correct frontmatter, all sections present |
| 3 | Document produced with minor formatting gaps |
| 2 | Incomplete document; missing key sections |
| 1 | No KIM document produced |

### 7. Next Steps (1-4)
Did Thomas suggest what to learn next?

| Score | Criteria |
|-------|----------|
| 4 | 2-3 specific suggestions with clear rationale for each |
| 3 | 2-3 suggestions without rationale |
| 2 | One generic suggestion |
| 1 | No next steps offered |

---

## Scoring Guide

| Total Score | Rating |
|-------------|--------|
| 25-28 | Excellent lesson |
| 20-24 | Good lesson |
| 14-19 | Needs improvement |
| 7-13 | Poor quality |

---

## Usage

Run this rubric when:
- Reviewing a sample lesson for calibration
- Evaluating Thomas outputs during testing
- Self-assessing a lesson before storing in KIM

Store evaluation results as SLG-NNNN in KIM with tag: eval
