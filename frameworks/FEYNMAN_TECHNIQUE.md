# The Feynman Technique

## What It Is
A learning method developed by physicist Richard Feynman. The core insight: if you cannot explain something in simple terms, you do not truly understand it.

## The Four Steps

### Step 1: Choose the Concept
Pick the topic you want to understand. Write it at the top of a blank page.

### Step 2: Explain It Simply
Write out an explanation as if you were teaching it to someone with no background - a curious 12-year-old, or a smart friend from a completely different field. Use plain language. No jargon.

### Step 3: Identify the Gaps
Where did you struggle to explain? Where did you reach for technical terms because you could not find simpler words? Those gaps reveal what you do not yet understand.

### Step 4: Return to the Source, Then Simplify Again
Go back to the material for the parts you could not explain. Study them. Then try explaining again - simpler, clearer, with better analogies.

## How Thomas Uses It
- Every lesson starts with the Big Idea in one plain sentence
- Thomas never uses jargon without immediately defining it
- Comprehension checks ask the user to explain back in their own words
- If the user cannot explain it simply, Thomas re-teaches before moving on

## Example Application
**Topic:** Neural networks

**Bad explanation (jargon-heavy):** "A neural network is a parameterized function approximator trained via stochastic gradient descent to minimize a loss function over a dataset."

**Feynman explanation:** "A neural network is a system that learns by trying, failing, and adjusting - like a toddler learning to walk. It makes a guess, sees how wrong it was, and tweaks its internal settings a tiny bit to do better next time. Do this millions of times and it gets very good."

## Why It Works
Understanding is not the same as recognition. You can read something and feel you understand it. The Feynman Technique forces you to generate an explanation, which reveals the difference between real understanding and the illusion of it.
